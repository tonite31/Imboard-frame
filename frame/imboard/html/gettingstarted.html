1. 다운로드

1-1. 사전준비

웹서버
임보드를 설치할 호스팅 서버 또는 개인 서버가 필요합니다.
nodejs & MySQL
임보드는 nodejs기반 플랫폼이며 데이터베이스는 MySQL을 사용합니다. 최신버전의 nodejs와 MySQL을 설치해주시기 바랍니다.
데이터베이스 및 사용자 생성
설치한 MySQL에 접속한 뒤, 사용자와 데이터베이스를 생성합니다.

1-2. 다운로드

임보드는 zip과 tar.gz형태로 배포됩니다. 최신버전은 입니다.

개발중인버전은 https://github.com/tonite31/Imboard/tree/development 에서 다운로드 받을 수 있습니다.

imboard.zip
#{lan.download} imboard.tar.gz
#{lan.download}

1-3. NPM으로 설치하기

npm을 이용하면 손쉽게 임보드를 설치할 수 있습니다.

$ npm install imboard

2. 설정 및 실행

임보드를 구동하기 위해 필요한 nodejs 모듈을 설치하고 설정값을 입력합니다. 이 작업은 임보드가 설치된 디렉토리에서 진행합니다.

nodejs 모듈 설치
$ npm install
데이터베이스 접속정보 설정
$ node config.js

jdbc메뉴를 선택한 후 host, username, password, database 정보를 차례로 입력합니다.

프레임 이름 설정
$ node config.js

frame메뉴를 선택한 후 적용할 프레임 이름을 입력합니다. 입력하지 않으면 기본값인 blog로 설정됩니다.

이 예제에서는 기본값인 blog로 입력합니다.

초기데이터입력
$ node install.js

위에서 설정한 데이터베이스로 임보드를 위한 초기데이터와 프레임을 위한 초기데이터가 입력되고 임보드 설치작업이 끝납니다.

어플리케이션 실행
$ node app.js

브라우저에 http://localhost:3000/을 입력하면 blog프레임으로 구동된 어플리케이션을 확인할 수 있습니다.

3. 프레임구조

프레임은 웹페이지 단위입니다. 디렉토리로 구성되며 /content/frame/ 경로 하위에 위치합니다.

다음 이미지는 기본 프레임 "blog"의 디렉토리 구조 입니다.

css
.css 파일들이 들어있는 디렉토리 입니다.
html
.html 파일들이 들어있는 디렉토리 입니다. index.html파일은 시작화면으로써 반드시 존재해야 합니다.
js
.js 파일들이 들어있는 디렉토리입니다.
lib
프레임에서 사용할 라이브러리들이 들어있는 디렉토리입니다.
properties
프레임에 필요한 초기데이터를 위한 데이터베이스 스크립트나 핸들바 헬퍼 스크립트, 국제화 데이터가 들어있는 디렉토리입니다. 하위의 handlebars와 setup디렉토리 이름은 변경되면 안됩니다.

4. index.html

index.html 파일은 시작화면이고 다음과 같은 구조를 가져야합니다. 프레임에 필요한 최소한의 자바스크립트 라이브러리와 css파일이 import되어 있습니다.


			
##{path.xxx}형태로 입력된 부분은 치환변수로 각각의 xxx키에 해당하는 경로로 치환됩니다. 자세한 내용은 문서의 치환변수를 참조하십시오.

		

		

			
5. UI프레임워크 임포트

			
Bootstrap과 같은 UI프레임워크를 사용할때는 프레임의 lib디렉토리에 복사하고 script와 css를 index.html 파일에 import합니다.

			

			
<link rel="stylesheet" href="##{path.root}/lib/bootstrap/css/bootstrap-theme.min.css"/>
<link rel="stylesheet" href="##{path.root}/lib/bootstrap/css/bootstrap.min.css"/>
<script src="##{path.root}/lib/bootstrap/js/bootstrap.min.js"></script>

		

		

			
6. 예제

			
이 예제에서는 간단한 게시판을 구축하는 방법을 소개합니다. 예제에서 제작한 프레임은 Sample을 통해 다운로드 받을 수 있습니다.

			

				
6-1. 프레임 디렉토리 생성

				
다음 이미지와 같이 sample 프레임 디렉토리를 생성합니다. sample 디렉토리는 /content/frame/ 디렉토리 하위에 위치해야합니다.

				

			

			

				
6-2. index.html

				
index.html 파일은 게시판 목록을 보여주고 글쓰기 버튼을 클릭하면 게시글을 작성할 수 있는 화면으로 전환됩니다.

				

				
#{lan.boardName} : 치환변수를 활용하여 다국어를 처리합니다. localize.json에 입력된 boardName값이 클라이언트의 언어에따라 치환됩니다.

				
<div data-fragment="body"> : 화면 조각이 보여지는 태그입니다. 글쓰기 버튼을 클릭하면 파라미터에 body=write가 입력되며 write.html의 내용이 이 태그 내부에 보여지게 됩니다.

				
data-bind="articleComponent" data-param='{"boardId" : "sample"}' : articleComponent모듈을 호출하며 data-param에 설정한 파라미터가 전달됩니다. sample이라는 boardId를 가지고 있는 게시글목록을 불러옵니다.

				
불러온 게시글 목록은 <script id="articleComponentTemplate" type="text/x-handlebars-template"></code> 핸들바 템플릿 스크립트 활용하여 <div data-parts="body" data-template-id="articleComponentTemplate"></div> 태그 내부로 보여줍니다.

				
자세한 정보는 문서의 데이터조합을 참고하십시오.

			

			

				
6-3. index.js

				
게시글 목록의 삭제 버튼을 클릭하면 해당 게시글을 삭제하는 기능을 구현합니다.

				

				
$.api.article.deleteArticle 함수에 대한 자세한 정보는 문서의 deleteArticle을 참고하십시오.

			

			

				
6-4. index.css

				

			

			

				
6-5. write.html

				
제목과 내용을 입력받아 게시글을 저장하는 화면입니다.

				

			

			

				
6-6. write.js

				

				
form submit이 발생하면 compile함수의 파라미터로 전달된 함수를 실행하며 data에 사용자가 입력한 값을 담아줍니다. 그 data를 사용해서 게시글 작성화면인경우 insertArticle을 호출하고 편집화면일경우 updateArticle을 호출합니다.

				
form 컴포넌트에 대한 자세한 정보는 문서의 form 컴포넌트를 참고하십시오.

2. Configuration and run

			
Install a nodejs module and configure a specific value for run a imboard application. this work is processed on a root directory of installed imboard.

			

				
Install a nodejs module

				
$ npm install

			

			

				
Configure information of database

				

					
$ node config.js

					
Select a jdbc menu. enter the information of host, username, password, database.

				

			

			

				
Configure a name of frame

				

					
$ node config.js

					
Select a frame menu. enter the name of frame. If you don't a name, setted "blog".

					
For this tutorial we will enter a "blog".

				

			

			

				
Install initial data

				

					
$ node install.js

					
Insert initial data for a imboard application and a frame to database of previous configuration.

					
Now, installation is done.

				

			

			

				
Run application

				

					
$ node app.js

					
Enter http://localhost:3000/ in a web browser. you can see application running by frame "blog".

				

			

		

		

			
3. Structure of frame

			
Frame is unit of webpage. it is composited by directory and it is placed under /content/frame/

			
Following image is a structure of default frame "blog".

			

			

				
css

				
This is a directory include .css files.

			

			

				
html

				
This is a directory include .html files. index.html file is a start page and essential.

			

			

				
js

				
This is a directory include .js fiels.

			

			

				
lib

				
This is a directory include a library for frame.

			

			

				
properties

				
This is a directory include a script of database for initial data for frame and a script of handlebars helper and a data of localization. do not change both directory name "handlebars" and "setup"

			

		

		

			
4. index.html

			
index.html file is a start page and has a following structure. it has a essential css files and javascript library for the frame.

			

			
##{path.xxx} is a replace variables that is replaced by specific path of xxx key. For more information about replace variables, see Replace variables.

		

		

			
5. Import UI-Framework

			
If you want to use UI-Framework like the "Bootstrap", copy the library into a directory "lib" of the frame and import script and css into "index.html".

			

			
<link rel="stylesheet" href="##{path.root}/lib/bootstrap/css/bootstrap-theme.min.css"/>
<link rel="stylesheet" href="##{path.root}/lib/bootstrap/css/bootstrap.min.css"/>
<script src="##{path.root}/lib/bootstrap/js/bootstrap.min.js"></script>

		

		

			
6. Example

			
This example describe how to develop a simple board. a frame which is developed in this example can download from Sample.

			

				
6-1. Create frame directory

				
Create a directory of frame "sample" refer to following image. "sample" directory is in "/content/frame/" directory.

				

			

			

				
6-2. index.html

				
index.html show a list of article. if you click a button "Write", change to a page which an write a article.

				

				
#{lan.boardName} : localization using replace variables. this is replaced to value of "boardName" in lcoalize.json

				
<div data-fragment="body"> : this tag show fragment. if you click a button "write", shown a content of "write.html" in this tag after url parameter is changed to body=write.

				
data-bind="articleComponent" data-param='{"boardId" : "sample"}' : call a module "articleComponent" with a parameter which set in data-param attribute. load a list of article which has a boardId "sample".

				
The list of article that is loaded previous step is binded into <div data-parts="body" data-template-id="articleComponentTemplate"></div> by <script id="articleComponentTemplate" type="text/x-handlebars-template"></code> handlebars template script.

				
For more information about that, see Data composition.

			

			

				
6-3. index.js

				
This script is developed to delete function, if you click a button "delete" in list of article.

				

				
For more information about $.api.article.deleteArticle, see deleteArticle.

			

			

				
6-4. index.css

				

			

			

				
6-5. write.html

				
This page save a article with a subject and a content by user input.

				

			

			

				
6-6. write.js

				

				
If submit event occured in a form element, call a function which is passed in function "compile". it has a data which is entered by user. if this page is write mode, call a function "insertArticle", otherwhise call a function "updateArticle" using the data.

				
For more informatin about form component, see form component.
