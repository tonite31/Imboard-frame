1. 개요

임보드 어플리케이션은 프레임단위로 실행됩니다. 시작하기에서는 프레임을 개발하는 방법에 대해 간략하게 소개합니다.

본 시작하기는 nodejs와 mysql이 이미 설치되어있고 초기 세팅이 완료되었다는 가정하에 시작합니다. 개발은 이클립스로 진행하며 시작하기를 통해 제작되는 웹페이지는 아래와 같습니다.

4. 레이아웃

가장먼저 header부분과 body부분으로 나뉘는 레이아웃을 작성합니다. data-fragment 속성이 지정된 div는 앞으로 body라고 부릅니다. body부분은 화면 조각들이 보여질 컨테이너 역할을 하게됩니다.

예를들어 URL에 ?body=boardList가 지정되면 2번에서 작성한 html폴더내부에 boardList.html을 불러와서 body 내부로 보여줍니다.

이 버튼을 클릭하면 게시판생성 화면조각을 불러와서 body부분에 보여줍니다.

html폴더에 addBoard.html 파일을 생성합니다. 이 파일은 조각파일로 완전한 html구조를 갖추지 않아도 됩니다.

게시판생성에 필요한 정보를 입력받기 위해 아래와 같은 코드를 작성합니다. 각 input태그에는 적절한 name속성을 지정하고 id와 같이 반드시 필요한 값은 required="required" 속성을 지정해서 validation 체크도 될 수 있게 합니다.

이 화면조각은 게시판을 생성함과 동시에 편집도 가능하게 하기 위해 아래와 같이 data-bind="board"속성을 지정합니다. data-param='{"id" : "#{id}"}'속성을 지정해주면 파라미터에 id값을 ##{id}부분에 치환해서 board모듈을 호출합니다. 만약 id에 해당하는 게시판데이터가 있다면 각 input태그에 값이 보여질것입니다.

form태그 코드 삽입

이제 각 input 태그에 입력된 정보를 가져와서 서버로 게시판 생성을 위한 요청을 보내야합니다. js폴더에 addBoard.js파일을 생성합니다. 화면조각 html파일과 동일한 이름으로 js파일을 생성하면 해당 html이 보여질때 js파일도 함께 import가 됩니다.

파라미터에 id값이 없다면 게시판을 생성하는 로직을 수행하고 id값이 있다면 게시판을 편집하는 로직을 수행하는 코드입니다.

js코드 삽입

form태그의 id를 활용해 위와같이 compile함수를 호출하면 form태그에 submit이 일어났을때 callback함수의 파라미터로 input태그에 입력된 값들을 json 형식으로 전달해줍니다. 이를 위해서는 반드시 각 input태그에 name속성이 지정되어있어야 합니다.

이 기능을 사용하기 위해서는 반드시 index.html파일에 imboard-ui.js파일이 import되어있어야 합니다.
