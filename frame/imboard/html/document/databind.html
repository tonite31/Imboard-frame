2. 서버데이터를 조합하는 방법

미리 정의된 치환변수를 사용하거나 HTML 엘리먼트에 data-bind속성을 지정하면 간편하게 서버로부터 데이터를 불러와서 화면에 조합할 수 있습니다.

2-1. 치환변수

치환변수를 사용하면 서버의 데이터를 간편하게 화면에 조합할 수 있습니다. 제공되는 치환변수는 path, query, user, lan이며 html파일에 ##{path.key} 형태로 입력합니다. 문자 그대로 사용하려면 #을 하나 더 붙인 ###{path.key} 형태로 입력합니다.

##{path.root}프레임의 루트경로입니다. 프레임의 이름이 sample이라면 치환되는 값은 /content/frame/sample 입니다.
##{path.module}프레임의 공통 모듈경로입니다. 치환되는 값은 /content/module 입니다.
##{path.lib}프레임의 공통 라이브러리경로입니다. 치환되는 값은 /content/lib 입니다.
##{query.xxx}xxx키에 해당하는 querystring값으로 치환됩니다. ##{query}만 지정할경우 ?를 포함한 querystring 전체가 치환됩니다.
##{user.xxx}로그인한 사용자의 정보중 xxx키에 해당하는 값으로 치환됩니다.
##{lan.xxx}Client의 언어에 따라 languages.json에 입력된 xxx키에 해당하는 값으로 치환됩니다.

2-2. data-bind

2-2-1. 개요

data-bind 모듈은 HTML 태그에 data-bind, data-param, data-template-id 속성을 지정해서 사용할 수 있습니다.

data-bind 속성값에 해당하는 모듈을 실행하고 데이터를 불러와서 Handlebars 문법을 통해 화면에 조합합니다.

<div data-bind="모듈명" data-param='{"key" : "value"}' data-template-id="핸들바 템플릿 아이디"></div>

2-3. Handlebars Helper

data-bind 모듈로 가져온 데이터는 Handlebars 문법으로 화면에 조합됩니다. 여기서는 추가적으로 구현된 Helper 문법을 소개하며 Handlebars의 기본적인 문법은 Handlebars 홈페이지를 참고하십시오.

equals 첫번째 파라미터와 두번째 파라미터가 일치하면 true, 일치하지 않으면 false
{{#equals param1 param2}} ... {{else}} ... {{/equals}}
notequals 첫번째 파라미터와 두번째 파라미터가 일치하지 않으면 true, 일치하면 false
{{#notequals param1 param2}} ... {{else}} ... {{/equals}}
empty 첫번째 파라미터가 빈값이면 true, 그렇지 않으면 false
{{#empty param}} ... {{else}} ... {{/equals}}
notempty 첫번째 파라미터가 빈값이 아니면 true, 그렇지 않으면 false
{{#notempty param}} ... {{else}} ... {{/equals}}
tojson 첫번째 파라미터가 JSON형태의 object이고 두번째 파라미터가 object의 key값일때 value를 리턴합니다.
{{tojson param1 param2}}
math 첫번째 파라미터와 세번째 파라미터를 두번째 파라미터로 연산한 결과를 리턴합니다.
{{math param1 "(+, -, *, /, %)" param3}}
sign 첫번째 파라미터와 세번째 파라미터를 두번째 파라미터로 비교한 결과를 리턴합니다.
{{sign param1 "(<, lt, <=, lt=, >, gt, >=, gt=, ==, ===, !=, !==)" param3}}
shortDate 첫번째 파라미터가 yyyy-mm-dd hh:mm:ss형식일때 오늘이면 hh:mm:ss형태로 오늘이 아니면 yyyy-MM-dd 형태로 리턴합니다.
{{shortDate param1}}
escape 첫번째 파라미터를 escape처리해서 리턴합니다.
{{escape param1}}
