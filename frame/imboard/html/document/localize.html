6. 국제화처리

치환변수 또는 {{##localize}} 문법을 활용하여 국제화처리를 할 수 있습니다.

6-1. 치환변수를 이용하는 방법

프레임 폴더 하위에 properties폴더를 생성합니다. properties폴더 하위에 localize.json파일을 생성합니다.

아래와 같은 구조로 localize.json 파일을 작성합니다.

{
	"signin" :
	{
		"ko-KR" : "로그인",
		"default" : "Sign in"
	},
	"signout" :
	{
		"ko-KR" : "로그아웃",
		"default" : "Sign out"
	}
}

localize.json을 위와 같이 작성하면 ##{lan.signin}, ##{lan.signout} 변수가 client의 언어에 따라 해당 값으로 치환됩니다. 만약 작성되지 않은 언어일경우 default에 입력한 값으로 치환됩니다.

6-2. {{##localize}} 문법을 이용하는 방법

{{##localize}}
	@#{ko-KR}
	한국어
	@#{en-US}
	Korean
	@#{default}
	Korean
{{//localize}}

{{##localize}} ~ {{//localize}}문법으로 감싼 부분에 각 언어에 해당하는 컨텐츠를 입력합니다.

@{default}

6. Localize usage

Can localize by replace variable or {{##localize}} grammar.

6-1. Replace variable usage

Create a "properties" directory in a "frame" directory. create a "localize.json" file in a "properties" directory.

Write a "localize.json" file refer to following structure.

{
	"signin" :
	{
		"ko-KR" : "로그인",
		"default" : "Sign in"
	},
	"signout" :
	{
		"ko-KR" : "로그아웃",
		"default" : "Sign out"
	}
}

##{lan.signin}, ##{lan.signout} is replaced to a value by locale of clinet. if locale is not writed in "localize.json" file, then replaced to a value of "default".

6-2. {{##localize}} usage

{{##localize}}
	@#{ko-KR}
	한국어
	@#{en-US}
	Korean
	@#{default}
	Korean
{{//localize}}

Write a content into "{{##localize}} ~ {{//localize}}". locale area that is equals to a locale of client is seen.

{{/localize}}
