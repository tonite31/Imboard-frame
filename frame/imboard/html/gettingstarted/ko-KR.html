1. 다운로드

1-1. 사전준비

웹서버
임보드를 설치할 호스팅 서버 또는 개인 서버가 필요합니다.
nodejs & MySQL
임보드는 nodejs기반 플랫폼이며 데이터베이스는 MySQL을 사용합니다. 최신버전의 nodejs와 MySQL을 설치해주시기 바랍니다.
데이터베이스 및 사용자 생성
설치한 MySQL에 접속한 뒤, 사용자와 데이터베이스를 생성합니다.

1-2. 다운로드

임보드는 zip과 tar.gz형태로 배포됩니다. 최신버전은 입니다.

개발중인버전은 https://github.com/tonite31/Imboard/tree/development 에서 다운로드 받을 수 있습니다.

imboard.zip
#{lan.download} imboard.tar.gz
#{lan.download}

1-3. NPM으로 설치하기

npm을 이용하면 손쉽게 임보드를 설치할 수 있습니다. 설치를 원하는 디렉토리에서 수행해 주십시오.

$ npm install imboard

2. 설정 및 실행

임보드를 구동하기 위해 필요한 nodejs 모듈을 설치하고 설정값을 입력합니다. 이 작업은 루트 디렉토리에서 진행합니다.

만약 윈도우를 사용중이라면 ls -al 대신 dir 명령어를 입력하십시오.

nodejs 모듈 설치

임보드 서버에 필요한 nodejs모듈을 설치해야합니다. 아래 명령어를 실행하면 루트 디렉토리에 node_modules 디렉토리가 생성됩니다.

$ npm install
데이터베이스 접속정보 설정
$ node config.js

jdbc메뉴를 선택한 후 host, username, password, database, port 정보를 차례로 입력합니다.

프레임 이름 설정

데이터베이스 접속정보 설정 이후 frame메뉴를 선택한 후 적용할 프레임 이름을 입력합니다. 입력하지 않으면 기본값인 sample로 설정됩니다.

여기서 프레임 이름을 다른걸로 변경하면 /content/frame/ 디렉토리 하위의 일치하는 프레임으로 어플리케이션이 구동됩니다.

이 예제에서는 기본값인 sample로 입력합니다.

포트설정

어플리케이션을 구동할 서버의 포트를 설정합니다. 기본값은 3000이며 만약 포트를 변경하려고 한다면 server메뉴를 선택한 후 포트를 입력하십시오.

이 예제에서는 기본값인 3000을 사용합니다.

초기데이터입력
$ node install.js

위에서 설정한 데이터베이스로 임보드를 위한 초기데이터와 프레임을 위한 초기데이터가 입력되고 임보드 설치작업이 끝납니다.

어플리케이션 실행
$ node app.js
$ node app.js -deploy

-deploy 옵션을 지정하면 console.log가 보여지지 않고 모든 로그는 logs디렉토리 내부에 쌓이게 됩니다.

브라우저에 http://localhost:3000/을 입력하면 sample프레임으로 구동된 어플리케이션을 확인할 수 있습니다.

글쓰기 버튼을 클릭하면 제목과 내용, 작성자를 입력할 수 있는 화면으로 전환되며 저장한 데이터는 목록에 표시됩니다.

로그인

임보드는 로그인페이지를 내장하고 있습니다. sample 프레임에서는 시작화면의 로그인 버튼을 클릭하면 로그인페이지로 이동됩니다.

로그인페이지는 다음 URL로 접근할 수 있습니다. http://localhost:3000/signin

관리자 계정으로 직접 로그인하거나 페이스북, 트위터, 구글의 OAuth를 활용하여 로그인을 할 수 있습니다. OAuth를 사용하기 위해서는 config.json파일의 OAuth정보를 입력해야합니다. 자세한 정보는 문서를 참조하십시오.

이 예제에서는 관리자 계정으로 직접 로그인합니다. 관리자 계정의 초기 아이디와 비밀번호는 다음과 같습니다. admin/admin

추후에 이메일을 활용한 회원등록 기능이 추가될 예정입니다.

관리자 대쉬보드

게시판, 메뉴와 같은 데이터를 쉽게 관리할 수 있는 관리자 대쉬보드가 준비되어있습니다. sample 프레임에서는 시작화면의 관리자 대쉬보드 버튼을 클릭하면 이동됩니다.

다음 URL로도 접근할 수 있습니다. http://localhost:3000/settings.

레벨이 0 미만인 관리자들만 대쉬보드에 접근할 수 있습니다.

게시판을 추가하거나 수정하면 시작화면의 게시판 목록에 반영됩니다.

각 메뉴별 사용법은 문서를 참고하십시오.

3. 프레임구조

이번 스텝에서는 프레임의 구조에 대해 소개합니다. 모든 프레임 디렉토리는 /content/frame/하위에 위치해야하며 아래에서 설명하는 구조를 갖춰야만 정상적으로 어플리케이션이 구동됩니다.

다음 이미지는 "sample" 프레임의 디렉토리 구조 입니다.

css
.css 파일들이 들어있는 디렉토리 입니다.
html
.html 파일들이 들어있는 디렉토리 입니다. index.html파일은 시작화면으로써 반드시 존재해야 합니다.
js
.js 파일들이 들어있는 디렉토리입니다.
lib
프레임에서 사용할 라이브러리들이 들어있는 디렉토리입니다.
properties
프레임에 필요한 초기데이터를 위한 데이터베이스 스크립트, 핸들바 헬퍼 스크립트, 국제화 데이터가 들어있는 디렉토리입니다.

4. index.html

index.html 파일은 시작화면입니다. 프레임에 필요한 최소한의 자바스크립트와 css파일이 import되어 있는 코드입니다.


			
##{path.xxx}형태로 입력된 부분은 현재 프레임의 경로로 변경되는 치환변수 입니다. 자세한 내용은 문서의 치환변수를 참조하십시오.

5-1. 프레임 디렉토리

				
다음 이미지는 sample프레임의 디레토리 입니다.

				

			

			

				
5-2. index.html

				
sample 프레임으로 구동될 웹페이지의 시작화면입니다. 게시판 목록을 보여주고 글쓰기 화면으로 이동할 수 있는 버튼이 있습니다.

				

				
#{lan.boardName} : 치환변수를 활용하여 다국어를 처리합니다. localize.json에 설정된 boardName값으로 클라이언트의 언어에따라 치환됩니다. localize.json파일은 5-7. localize.json을 참고하십시오.

				
<div data-fragment="body"> : 화면 조각이 보여지는 태그입니다. 글쓰기 버튼을 클릭하면 write.html의 내용이 이 태그 내부에 보여지고 js디렉토리의 write.js와 css디렉토리의 write.css파일이 자동으로 import 됩니다.

				
data-bind="articleComponent" data-param='{"boardId" : "sample"}' : articleComponent모듈을 호출하여 boardId가 sample인 게시글 목록을 불러옵니다.

				
불러온 게시글 목록은 <script id="articleComponentTemplate" type="text/x-handlebars-template"></code> 핸들바 템플릿 스크립트 활용하여 <div data-parts="body" data-template-id="articleComponentTemplate"></div> 태그 내부로 보여집니다.

				
articleComponent모듈은 서버로부터 페이징처리된 게시글 목록을 불러오는 모듈입니다. 데이터바인드 모듈에 대한 자세한 정보는 문서의 데이터조합을 참고하십시오.
