1. Download and installation

1-1. Preparation

Web server
You need a hosting server or private server for installing Imboard.
nodejs & MySQL
Imboard use the nodejs and MySQL database. Please Install recent version nodejs and MySQL.
Create a database and a user
Create database and user after connect to installed MySQL.

1-2. Download

Imboard is provided by .zip , .tar and .gz file formats. Currently version is .

Development version is provided from https://github.com/tonite31/Imboard/tree/development.

imboard.zip
#{lan.download} imboard.tar.gz

1-3. Install with npm

If you use "npm", you can install imboard easily.

$ npm install imboard

2. Configuraion and run

Install nodejs modules, configure imboard application. Following step is processed on root directory.

Install nodejs modules

You must to install nodejs modules for imboard server. Run following command, created a directory "node_modules" in root directory.

$ npm install
Configure database connection
$ node config.js

Select a menu "jdbc", enter a information host, username, password, database.

Set a frame's name

Select a menu "frame", enter a frame's name after configure database connection. If you not enter a specific value, setted a default value "sample".

If you change a frame's name to someting, ran application by a "someting" frame in /content/frame/ directory.

This example is entered "sample".

Enter a initial value
$ node install.js

Inserted a initial data for imboard and frame to database that you set a information in previous step. Now imboard installation work is done.

Run a application
$ node app.js
$ node app.js -deploy

If you set a option "-deploy", all console.log is not printed on shell, all log is stacked in logs directory.

Visit a url http://localhost:3000/ in a web browser. You can see a application that ran by frame "sample".

If you click a button "Write a article", moved to "write" page that can enter a subject and a content, a writer's name. That data is shown on the list.

3. Structure of frame

This step is introduce a structure of frame. All frame's directory must placed under the /content/frame/ directory and it muse have a following structure.

Following image is structure of frame "sample".

css
This directory include .css files.
html
This directory include .html files. index.html is main page, so it must be created.
js
This directory include .js files.
lib
This directory include libraries that used for the frame.
properties
This directory include database script for initial data that is needed for frame, handlebars helper script, localization data.

5-2. index.html

				
This is a main page of web page that is ran by frame "sample". This page is show a list of board and has a button that can move to "write" page.

				

				
#{lan.boardName} : is process a localization using replace variable. This is replaced by boardName that is set in "localize.json" by language of client. For more information, see 6. Localization

				
<div data-fragment="body"> : is a tag that shown page fragment. If you click a button "Write a article", show a content for "write.html" in this tag and imported a "write.js" in js directory and a "write.css" in css directory.

				
data-bind="articleComponent" data-param='{"boardId" : "sample"}' :  call "articleComponent" module, load a list of article that has a boardId "sample".

				
Loaded a list of article is shown into <div data-parts="body" data-template-id="articleComponentTemplate"></div> tag using Handlebars template script "<script id="articleComponentTemplate" type="text/x-handlebars-template"></code>"

				
articleComponent module load a list of article that is proccessed paging from the server. For more information of databind moduels, see data-bind modules.
